### **SAS Token (Query Params):**
```
?sv=2021-06-08                                    // Storage version
&st=2026-03-17T00%3A00%3A00Z                      // Start time (URL encoded)
&se=2026-03-17T01%3A00%3A00Z                      // End time (URL encoded)
&sr=b                                              // Resource type: blob
&sp=r                                              // Permissions: read
&sig=ABC123XYZ                                     // Signature
```

## ⚠️ **Notas Importantes:**

### **1. Tiempo de Espera:**
- El endpoint **ESPERA 5-15 minutos** antes de responder
- No hay timeout - debes esperar a que VEO3 termine
- La conexión HTTP se mantiene abierta

### **2. SAS Token:**
- Temporal (típicamente 1 hora)
- Solo lectura (`sp=r`)
- Específico para ese blob
- Se genera dinámicamente

### **3. Blob Storage:**
- Container: `videos`
- Formato: `.mp4`
- Calidad: 720p (VEO3 default)
- Tamaño: ~5-20 MB (dependiendo duración)

## 📄 **Código que Genera este JSON:**

### **En VeoVideoService.generateVideo():**
```typescript
async generateVideo(dto): Promise<{ videoUrl: string; filename: string }> {
  // 1. Llamar a VEO3 API
  const operation = await this.callVeoApiWithSdk(dto);
  const result = await this.pollForCompletion(operation);
  
  // 2. Extraer video bytes
  const videoBytes = this.extractVideoData(result);
  
  // 3. Guardar temporalmente
  const filename = `veo-video-${Date.now()}.mp4`;
  const tempPath = path.join(__dirname, '..', '..', '..', 'temp', filename);
  fs.writeFileSync(tempPath, videoBytes);
  
  // 4. Subir a Azure Blob con SAS
  const blobUrl = await this.azureBlobService.uploadFileToBlobWithSas(
    tempPath,
    `videos/${filename}`,
    'video/mp4',
  );
  
  return {
    videoUrl: blobUrl,  // ✅ URL completa con SAS token
    filename,
  };
}
```
